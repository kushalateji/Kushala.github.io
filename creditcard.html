Credit Card Fraud Detection using Machine Learning

Introduction

In response to the surge in online fraud during the COVID-19 pandemic, exacerbated by increased online shopping, a model is proposed to combat credit and debit card fraud. Leveraging machine learning, particularly logistic regression, this model aims to distinguish between legitimate and fraudulent transactions, providing a binary classification (TRUE or FALSE) result.

Key Aspects

  • Machine Learning Approach:
    • Utilizes supervised learning, specifically logistic regression, for binary classification of transactions.
    • Trains the model on a dataset, including both fraudulent and legitimate transactions, obtained from Kaggle.
  • Data Imbalance Handling:
    • Addresses the imbalanced dataset, with only 492 fraudulent transactions out of 284,807, by employing under-sampling to create a balanced dataset.
  • Feature Transformation:
    • Employs Principal Component Analysis (PCA) to convert sensitive information into numerical values, ensuring privacy protection.

Experimental Results

  • Model Accuracy:
    • Achieves approximately 92% accuracy on training data and 90% on testing data, indicating robust performance.

Algorithm Implementation

  • Logistic Regression:
    • Applies logistic regression for binary classification, providing results between 0 and 1.
    • Steps include data pre-processing, model fitting, result prediction, confusion matrix creation, and visualization of test set results.

Training and Testing Dataset

  • Data Division:
    • Divides the dataset into training (80%) and testing (20%) sets, with under-sampling to balance fraudulent and legitimate transactions.

Realistic Constraints

  • Imbalanced Dataset:
    • Faces challenges due to a significantly higher number of legitimate transactions compared to fraudulent ones.
    • Deals with transformed numerical data using PCA for privacy reasons.

Coding and Testing

  • Libraries Used:
    • Imports necessary libraries such as NumPy and pd for code functionality.

Result

  • Code Output:
    • Achieves an accuracy of around 90%, demonstrated through output screenshots.
